# LogTail 快速使用指南

## 🚀 快速启动

### Windows 用户
双击运行 `start.bat` 文件即可。

### 手动启动
```bash
cd c:\Coding\logtail
python server.py
```

启动后访问：http://localhost:8080

## 📋 功能说明

### 1. 选择服务器
- 在页面顶部下拉菜单中选择要连接的服务器
- 服务器列表来自 `config.json` 配置

### 2. 浏览目录
- 左侧面板显示服务器上的 `/data/logs` 目录
- 点击 📁 目录图标可以展开查看子目录
- 点击 📄 文件图标可以选择该日志文件

### 3. 实时监控
- 选择文件后，点击"▶ 开始监控"按钮
- 右侧面板会实时显示日志内容
- 日志会自动滚动到最新行
- 点击"⏹ 停止监控"停止查看

### 4. 日志高亮
系统自动识别并高亮显示：
- 🔴 **红色** - ERROR 级别日志
- 🟡 **黄色** - WARN 级别日志  
- 🔵 **蓝色** - INFO 级别日志

## ⚙️ 配置说明

### 配置文件位置
`c:\Coding\logtail\config.json`

### 配置示例
```json
{
  "servers": [
    {
      "id": "server1",
      "name": "应用服务器 1",
      "host": "192.168.0.10",
      "port": 22,
      "username": "root",
      "password": "你的密码",
      "base_path": "/data/logs"
    }
  ]
}
```

### 配置项说明
- `id`: 服务器唯一标识
- `name`: 服务器显示名称
- `host`: 服务器 IP 地址
- `port`: SSH 端口，默认 22
- `username`: SSH 用户名
- `password`: SSH 密码（如无密码留空）
- `key_file`: SSH 私钥文件路径（如使用密钥认证）
- `base_path`: 日志文件基础路径

## 🔐 认证方式

### 密码认证
在配置文件中填写 `password` 字段：
```json
{
  "username": "root",
  "password": "your_password"
}
```

### 密钥认证
在配置文件中填写 `key_file` 字段：
```json
{
  "username": "root",
  "key_file": "C:/Users/YourName/.ssh/id_rsa"
}
```

## ⚠️ 常见问题

### 1. 无法连接服务器
**原因**：SSH 连接失败
**解决**：
- 检查服务器 IP 是否正确
- 确认 SSH 服务是否运行
- 验证用户名和密码
- 检查防火墙设置

### 2. 目录加载失败
**原因**：路径不存在或无权限
**解决**：
- 确认 `/data/logs` 目录存在
- 检查 SSH 用户是否有读取权限
- 查看后端日志获取详细错误

### 3. 日志不更新
**原因**：文件没有新内容写入
**解决**：
- 确认应用正在写入日志
- 检查日志文件路径是否正确
- 查看 WebSocket 连接状态

## 🛠️ 高级用法

### 修改默认端口
编辑 `server.py`，修改启动端口：
```python
web.run_app(app, host='0.0.0.0', port=8080)  # 修改 port 参数
```

### 添加更多服务器
在 `config.json` 中添加新的服务器配置即可。

### 自定义日志路径
修改配置文件中的 `base_path` 字段。

## 📝 技巧

1. **快速定位错误**：关注红色高亮的 ERROR 日志
2. **多标签页**：可以打开多个浏览器标签页监控不同文件
3. **清空显示**：点击"🗑 清空显示"按钮清除当前日志
4. **状态栏**：底部状态栏显示当前行数和连接状态
